# Convenciones:
Las convenciones en programación se refieren a un conjunto de reglas y prácticas aceptadas y utilizadas por los desarrolladores para escribir código de manera consistente y legible. Estas convenciones no son requisitos obligatorios del lenguaje de programación en sí, sino más bien recomendaciones o estándares adoptados por la comunidad de desarrollo para facilitar la comprensión y el mantenimiento del código.

Las convenciones abarcan diversos aspectos del desarrollo de software, como la nomenclatura de variables, funciones y clases, el formato del código, el manejo de errores, los comentarios, entre otros. Siguiendo estas convenciones, los desarrolladores pueden escribir código que sea más fácil de entender y mantener, tanto para ellos mismos como para otros miembros del equipo de desarrollo.

Además de mejorar la legibilidad y la comprensión del código, las convenciones también pueden ayudar a reducir errores, facilitar la colaboración entre desarrolladores y promover una estructura coherente en el código base de un proyecto.


    1 Nombres de Variables y Funciones:
        Utilizar camelCase para nombres de variables y funciones: 
            miVariable, miFuncion.
        Utilizar nombres descriptivos que reflejen el propósito de la variable o función.
    
    2 Constantes:
        Utilizar MAYÚSCULAS para nombres de constantes.: 
            PI, COLOR_PRINCIPAL.
    
    Comentarios:
        Utilizar comentarios para explicar el propósito y el funcionamiento del código.
        Mantener los comentarios actualizados si se realizan cambios en el código.
        Evitar comentarios obvios que simplemente repitan lo que hace el código.
    
    Espacios en Blanco:
        Utilizar espacios en blanco para mejorar la legibilidad del código.
        Separar las diferentes secciones del código con líneas en blanco.
        Utilizar indentación consistente (generalmente de 2 o 4 espacios) para organizar el código en bloques.
    
    Longitud de Línea:
        Limitar la longitud de línea a 80-100 caracteres para mejorar la legibilidad.
        Dividir líneas largas en líneas más cortas utilizando la continuación de línea apropiada.
   
    Declaración de Variables:
        Declarar todas las variables al principio de la función o bloque de código.
        Evitar la declaración global de variables siempre que sea posible.
    
    Uso de Punto y Coma:
        Utilizar punto y coma al final de cada declaración de línea para evitar problemas de interpretación del código.
    
    Comparaciones:
        Utilizar el operador === para comparar valores y tipos de datos.

    Estructuras de Control:
        Utilizar llaves {} incluso para bloques de código de una sola línea para mejorar la claridad y evitar errores.
    
    Nombres de Clases y Constructores:
        Utilizar PascalCase para nombres de clases y constructores. Por ejemplo: MiClase, MiConstructor.
    
    Nombres de Métodos:
        Utilizar camelCase para nombres de métodos dentro de una clase. Por ejemplo: miMetodo.
    
    Gestión de Errores:
        Utilizar bloques try-catch para capturar y manejar excepciones de forma adecuada.
        Evitar el uso excesivo de excepciones para controlar el flujo del programa.ados.
